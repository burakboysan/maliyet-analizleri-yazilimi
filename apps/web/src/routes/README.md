# Routes

TanStack Router uses file-based routing. Every `.tsx` file in this directory defines a route. Do not create `src/pages/`, `src/routes/_app/index.tsx`, or `app/layout.tsx`; those are Next.js / Remix conventions. The only root layout is `src/routes/__root.tsx`.

## Conventions

| File | URL |
| --- | --- |
| `index.tsx` | `/` |
| `users/index.tsx` | `/users` |
| `users/$id.tsx` | `/users/:id` dynamic segment |
| `posts/{-$category}.tsx` | `/posts/:category?` optional segment |
| `files/$.tsx` | `/files/*` splat segment, read via `_splat` |
| `_layout.tsx` | layout route, renders children via `<Outlet />` |
| `__root.tsx` | app shell, wraps every page and preserves `<Outlet />` |

`routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin.
