# Supabase Migration Runbook

Bu doküman, `urun_maliyet_db` MySQL verisini Supabase Postgres'e taşımak için uygulanacak güvenli akışı tanımlar.

## Ön Koşullar

- VM sadece dump almak için açılır.
- Dump alınmadan production verisine yazma işlemi yapılmaz.
- Dump dosyası checksum ile doğrulanır ve GCS'e yüklenir.
- İlk deneme Supabase staging database üzerinde yapılır.

## Dump Alma

VM açıldıktan sonra MySQL dump alınacak operasyon:

```bash
mysqldump --single-transaction --default-character-set=utf8mb4 urun_maliyet_db | gzip > urun_maliyet_db_$(date -u +%Y%m%dT%H%M%SZ).sql.gz
sha256sum urun_maliyet_db_*.sql.gz > urun_maliyet_db.sha256
```

Dump ve checksum GCS'e yüklenir. VM işlem sonunda kapatılır.

## Schema Conversion

MySQL -> Postgres dönüşümünde dikkat edilecek noktalar:

- `AUTO_INCREMENT` -> `GENERATED BY DEFAULT AS IDENTITY`
- `DATETIME` / `TIMESTAMP` -> `timestamp`
- `TINYINT(1)` -> `smallint`; boolean anlamı olan kolonlar uygulama katmanında ayrıca doğrulanır.
- `DECIMAL` hassasiyeti korunur.
- `utf8mb4` metinler UTF-8 olarak doğrulanır.
- MySQL engine/collation satırları kaldırılır.

Tekrarlanabilir dönüşüm için:

```powershell
python scripts\mysql_dump_to_postgres.py `
  outputs\supabase_migration\urun_maliyet_db_maliyet-server_20260626T084159Z.sql.gz `
  outputs\supabase_migration\supabase_import_20260626T084159Z.sql
```

Supabase Management API dosya boyutu limitine takılmamak için SQL dosyası chunk dosyalarına bölünür:

```powershell
python scripts\split_sql_for_supabase_query.py `
  outputs\supabase_migration\supabase_import_20260626T084159Z.sql `
  outputs\supabase_migration\supabase_import_20260626T084159Z_chunks `
  --max-bytes 200000
```

Linked Supabase projesine chunk dosyaları sırayla uygulanır:

```powershell
$chunks = Get-ChildItem -LiteralPath "outputs\supabase_migration\supabase_import_20260626T084159Z_chunks" -Filter "*.sql" | Sort-Object Name
foreach ($chunk in $chunks) {
  supabase db query --linked --file $chunk.FullName
  if ($LASTEXITCODE -ne 0) { exit $LASTEXITCODE }
}
```

Alternatif olarak, küçük SQL dosyaları Supabase Postgres connection string ile tek seferde import edilebilir:

```powershell
supabase db query --db-url "<SUPABASE_DATABASE_URL>" --file outputs\supabase_migration\supabase_import_20260626T084159Z.sql
```

`psql` kurulu olan makinelerde alternatif:

```powershell
psql "<SUPABASE_DATABASE_URL>" -v ON_ERROR_STOP=1 -f outputs\supabase_migration\supabase_import_20260626T084159Z.sql
```

## Doğrulama

Her import sonrası:

- Kritik tablolarda row count karşılaştırılır.
- Örnek kullanıcı, ürün, malzeme ve sihirbaz kayıtları kontrol edilir.
- `ğ, ü, ş, ı, ö, ç, İ` içeren kayıtların bozulmadan geldiği doğrulanır.
- API staging ortamında `/ready`, `/auth/login`, `/modules`, `/products`, `/materials` smoke testleri çalıştırılır.

## Kabul Kriteri

- Staging import tekrar çalıştırılabilir.
- Kritik tablolarda row count eşleşir.
- Türkçe karakter testleri geçer.
- Production import öncesi rollback dump'ı hazırdır.
